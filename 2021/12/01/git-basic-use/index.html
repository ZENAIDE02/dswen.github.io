git 常用命令总结
dongshiwen
  2021-12-01 16:14:25 2021-12-01 16:14

目的

为了方便之后用 github 托管项目，整理了一些 git 基础的使用场景，后续可继续补充添加。

github 更新

为避免带有“种族歧视色彩”，从2020年10月1日起， 在 github 上创建的所有新的源代码仓库将默认被命名为 “main”，而不是原先的”master”。

Git 本地项目与远端连接

  1. 本地新建 repo。
  2. repo 文件夹中使用 git init 命令，之后文件夹中多出来一个 .git 文件夹。

若之前与其他的远端建立关联，现在想断开，有两种方法。

  • Git Bash Here 输入
  • 直接删除本地库 .git 文件夹。
  1. 关联远程库。
  2. 将文件添加到暂存区。
  3. 将文件提交至仓库
  4. git pull 获取远程库与本地合并
  5. 将本地内容推送至远程

Git 克隆远端

  1. 克隆远端项目至本地
  2. 后续添加至暂存区、远端提交等操作相同。

Git 新建分支

  1. 克隆项目，若想开发新需求，新建分支。
  2. 查看分支状态
  3. 切换到本地新分支
  4. 提交至远端仓库
    (之后提交前要先拉取)
